# Visual Aids Site — Specification

## Overview

A GitHub Pages static site that hosts a collection of interactive React visual aids. Each visual aid lives in its own subfolder under `src/` and is accessible at its own URL. A root index page lists all available aids with rich metadata.

## Repository Structure

```
hkfosschat-visual-aids/
├── package.json
├── vite.config.js                    ← includes Vite plugin for multi-page entry discovery
├── index.html                        ← index page HTML shell
├── generated/                        ← gitignored; per-aid HTML shells written here at build time
│   └── 20260427-c-string/
│       └── index.html
├── src/
│   ├── index.jsx                     ← index page React component
│   ├── 20260427-c-string/
│   │   └── index.jsx                 ← visual aid component (default export + meta named export)
│   └── <YYYYMMDD>-<slug>/
│       └── index.jsx
└── .github/
    └── workflows/
        └── deploy.yml                ← CI/CD: build + publish to gh-pages branch
```

## Visual Aid Convention

Each visual aid is a subfolder of `src/` named `<YYYYMMDD>-<slug>` containing a single `index.jsx`.

`index.jsx` must have two exports:

```js
// Named export — metadata for the index page
export const meta = {
  title: 'C String Memory Visualizer',          // required
  description: 'Visualize how strcpy, strncpy, and memcpy handle null-terminated strings.',  // required
  tags: ['c', 'memory', 'security'],             // optional
};

// Default export — the React component
const App = () => { ... };
export default App;
```

`title` and `description` are required. `tags` is optional. If `title` is absent, it falls back to the slug-derived title (see below). Date is always derived from the folder name — it is not part of `meta`.

No other configuration is required to add a new aid; discovery is fully automatic.

## Slug-to-Date Derivation

The date is always derived from the folder name:

| Folder name              | Date       |
|--------------------------|------------|
| `20260427-c-string`      | 2026-04-27 |
| `20260501-tcp-handshake` | 2026-05-01 |

**Rules:**
1. First 8 characters are the date in `YYYYMMDD` format → formatted as `YYYY-MM-DD`.
2. The remainder (after the first hyphen following the date) is the slug portion.

**Fallback title** (used when `meta.title` is absent):
- Slug hyphens replaced with spaces, result title-cased. E.g. `c-string` → `C String`.

## Build System

**Tooling:** Vite + React

### Vite plugin (inside `vite.config.js`)

Replaces the separate `gen-entries.js` script. Runs at Vite config evaluation time:

1. Scans `src/` for subdirectories matching `[0-9]{8}-*`.
2. For each discovered aid, writes a minimal `generated/<slug>/index.html` that mounts `../../src/<slug>/index.jsx` into `<div id="root">`.
3. Builds `rollupOptions.input`:
   ```js
   {
     main: 'index.html',
     '20260427-c-string': 'generated/20260427-c-string/index.html',
     // ...
   }
   ```
4. Sets `base` to `/<repo-name>/` so asset paths resolve correctly on GitHub Pages.

`generated/` is gitignored.

### Per-aid HTML shell

Generated by the Vite plugin into `generated/<slug>/index.html`. Minimal structure:

```html
<!doctype html>
<html lang="en">
  <head><meta charset="UTF-8" /><title></title></head>
  <body>
    <div id="root"></div>
    <script type="module" src="../../src/<slug>/index.jsx"></script>
  </body>
</html>
```

The component's `meta.title` is used to set `<title>` at build time via the plugin.

### Index page (`src/index.jsx`)

Uses `import.meta.glob` to discover all aids at build time — no generated JSON file needed:

```js
const modules = import.meta.glob('./**/index.jsx', { eager: true });
// Each module has: module.default (component), module.meta (metadata)
// Key shape: './20260427-c-string/index.jsx' → parse slug and date from key
```

Renders a card grid sorted by date descending. Each card shows:
- Title (from `meta.title`, fallback to slug-derived title)
- Date (derived from folder name)
- Description (from `meta.description`)
- Tags (from `meta.tags`, if present)
- Link to `/<repo-name>/<slug>/`

## Deployment

**GitHub Actions workflow (`.github/workflows/deploy.yml`):**

Triggers on push to `main`.

Steps:
1. Checkout repo.
2. `npm ci`
3. `npm run build` (Vite plugin handles entry generation inline — no separate script step)
4. Deploy `dist/` to `gh-pages` branch via `peaceiris/actions-gh-pages`.

GitHub Pages is configured to serve from the `gh-pages` branch.

## URL Structure

| Page         | URL                                                  |
|--------------|------------------------------------------------------|
| Index        | `https://<user>.github.io/<repo>/`                   |
| C String aid | `https://<user>.github.io/<repo>/20260427-c-string/` |

## Adding a New Visual Aid

1. Create `src/<YYYYMMDD>-<slug>/index.jsx` exporting `meta` and a default React component.
2. Commit and push to `main`.
3. CI runs automatically — the aid appears on the index and gets its own page.

No manual changes to `vite.config.js` or any other config file are needed.
